# WORKOUT PLAN / SCHEDULE (Upcoming Tab & Calendar Integration)
- GET   /api/workout-plan
- POST  /api/workout-plan
- POST  /api/workout-plan/suggest
  * NOTE: The backend must be able to return the user's active workout plan and schedule.
  * The system should support three modes of scheduling:
    1. INFERRED: Auto-generated based on the user's historical workout logs and patterns.
    2. MANUAL: Explicitly created/updated by the user (via POST /api/workout-plan).
    3. AI-SUGGESTED: Generated by the AI based on the user's profile and goals (via POST /api/workout-plan/suggest).
  * This data will eventually have its own isolated tab in the dashboard.
  * Ensure this schedule integrates cleanly with the `GET /api/calendar/schedule` endpoint so that planned workouts appear directly on the user's calendar when it goes live.
