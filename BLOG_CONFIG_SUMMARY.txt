   - Adds each blog post URL to sitemap with priority 0.7, changefreq monthly

Files Created:
--------------
1. blog-post.html — Individual blog post detail page (/blog/:slug)
   - info-shell/info-nav layout matching terms page
   - Two-column: sidebar (nav, tags, published date, author) + article
   - article header with info-hero and info-kicker
   - Featured image section (hidden until loaded)
   - blog-post-content-section for rendered Markdown
   - Back to Blog navigation link
   - JSON-LD Article schema (populated by JS with actual post data)
   - Og:type "article" meta tags
   - legal-footer matching terms page

2. src/blog-post.js — Blog post detail page logic
   - Parses slug from window.location.pathname (/blog/:slug)
   - Fetches single post: GET /api/blog/posts/:slug
   - Renders Markdown to HTML (headers, bold, italic, code, links, images, lists, hr)
   - Updates JSON-LD Article schema: headline, description, datePublished,
     dateModified, author (Person), image, publisher (Organization)
   - Updates document.title and meta tags: description, og:title, og:description,
     og:url, og:image, canonical URL
   - Populates sidebar: tags, published date, author
   - Shows featured image if available

3. scripts/fetch-blog-posts.mjs — Pre-build blog post fetcher
   - Fetches all posts from https://api.thetrackerapp.io/api/blog/posts
   - Paginates through all pages (limit=50 per page)
   - Writes posts index to public/blog-posts.json
   - Writes individual posts to public/blog-posts/{slug}.json
   - Run via: npm run blog:fetch

4. public/blog-posts/.gitkeep — Directory placeholder for source control


Backend Endpoints Used:
-----------------------
   GET  /api/blog/posts          — List posts (pagination, tag, search)
   GET  /api/blog/posts/:slug    — Single post by slug
   POST /api/blog/posts          — Create post (admin only)
   GET  /api/user/role           — Check if user is admin

   Key field needed for SEO: "updatedAt" on posts (falls back to "publishedAt")


SEO Improvements Summary:
------------------------
   1. meta name="robots" content="index, follow"
   2. meta name="twitter:card" content="summary_large_image"
   3. og:image:width + og:image:height specified
   4. JSON-LD Blog schema on /blog (dynamic BlogPosting items from API)
   5. JSON-LD Article schema on /blog/:slug (dynamic from API)
   6. Canonical URLs on both pages
   7. Individual blog posts included in sitemap.xml
   8. Pre-build script stores posts for SEO crawlers (public/blog-posts.json)
   9. Dynamic og:title, og:description, og:image from post data


Build Workflow:
--------------
   npm run blog:fetch     # Fetch posts from backend, store for SEO
   npm run build          # Vite build (includes blog.html + blog-post.html)
   npm run sitemap:build  # Generate sitemap with blog post URLs
